Introduction

SvLis is a kernel set-theoretic geometric modeller. Details of how to use it can be found in the SvLis Book and the SvLis User Manual. This document describes svLis's distribution and installation.

SvLis is a library written in C++, and it is free, though it is copyrighted by The University of Bath and Information Geometers Ltd. You may not sell svLis, nor may you sell any code or programs incorporating svLis. If you need the source code for the core of svLis immediately, or if you wish to use svLis commercially, have a look at the svLis commercial licence. SvLis is not in the public domain.

The free svLis is distributed as a compiled library, a set of .h files, some user-modifiable procedures in source code (which form an ancilliary library), and the source code for test programs and utilities.

SvLis currently runs on PCs under Linux using the Gnu g++ compiler and under Microsoft Windows 95/8 and Windows NT using Visual C++, on Suns also using the Gnu compiler, and on Silicon Graphics computers using SG's own C++ compiler, CC, and on Apple Macs using the Metrowerks CodeWarrior Pro 4 compiler.


 
 

Obtaining SvLis

When you download svLis, we would appreciate it if you would send us your email address (though this is not a prerequisite of taking a copy).

Your address will not be forwarded to anyone else; we will merely use it to keep you abreast of updates. If at any time you wish to remove yourself from the list email svlis with the word `remove' in the body of the message.

As distributed, svLis consists of a compressed archive containing the whole of the the system. SvLis can be obtained by clicking on:

  • Linux svLis (2MB, compiled under Linux 2.0.36, RedHat 5.2),
  • Sun svLis (1.5 MB, compiled under Unix System V Release 4.0), or
  • Silicon Graphics svLis (2.3 MB, compiled under IRIX 6.5).
  • Microsoft Windows svLis (1.5MB, compiled under Visual C++ version 5.0).
  • Apple Mac svLis (1.3 MB, compiled under Metrowerks CodeWarrior Pro 4).
These files were updated on 14 April 1999.  SvLis version N (currently N=3) will always read svLis files from versions N and N-1. An update program is supplied with this release to update files to this version number.

You will also need a copy of the OpenGL graphics library. If you are using Microsoft Visual C++ or are on a Silicon Graphics, then you will already have this. For Suns, Macs and Linux you will have to obtain a copy. The free Mesa OpenGL-compatible library for Suns Linux and other platforms is available from here. For details of how to obtain OpenGL for the Mac see that link and the file README_MAC in the svLis distribution. For all platforms you will also need the equally free OpenGL GLUT (Graphics Library User Toolkit) library (this is included in the Mesa demonstration files, which you should also download if you're using Mesa).
